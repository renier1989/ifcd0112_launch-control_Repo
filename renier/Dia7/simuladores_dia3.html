Practica usuarios, grupos y permisos

Sin miedo a romper nada real — todo es simulación en el navegador

① Identificar usuario — whoami, id, groups

Objetivo: Aprender los comandos que te dicen quién eres para el sistema. Completa los 4 retos de la derecha escribiendo los comandos en el prompt. Tienes sugerencias abajo.
alumno@ubuntu:~$
# Escribe un comando o usa los botones de abajo. Prueba: whoami
alumno@ubuntu:~$

🎯 RETOS

  1. Averigua tu nombre de usuario
  2. Lista tu UID, GID y todos los grupos
  3. Ver los miembros del grupo sudo
  4. Encuentra tu entrada en /etc/passwd

② Crear usuario — adduser, usermod, deluser

Objetivo: Practica el ciclo completo de un usuario: crear → añadir a grupo → modificar → eliminar. Como en este simulador sí tienes permisos, cualquier sudo lo acepta directamente.
alumno@ubuntu:~$
# Crea un usuario de prueba. Empieza con: sudo adduser dev_test
alumno@ubuntu:~$

USUARIOS DEL SISTEMA

🎯 RETOS

  1. Crea el usuario dev_test
  2. Añádelo al grupo sudo (con -aG)
  3. Verifica sus grupos con id dev_test
  4. Bórralo junto con su home

③ su vs sudo — escalada de privilegios

Objetivo: Entender la diferencia entre cambiar de identidad (su) y ejecutar como root un comando (sudo). Observa quién eres en cada momento en el prompt.
Pista: Después de su - dev_test, fíjate cómo cambia el nombre del prompt. Prueba sudo apt update como dev_test para ver el error de "no está en sudoers".
alumno@ubuntu:~$
# Empieza como 'alumno'. Prueba: sudo whoami
alumno@ubuntu:~$

¿Es root?no

¿En grupo sudo?sí

🎯 RETOS

  1. Ejecuta un comando como root con sudo
  2. Cambia de sesión a dev_test con su -
  3. Intenta sudo apt update como dev_test (fallará)
  4. Vuelve a tu sesión con exit
  5. Lee /etc/shadow con sudo (solo root puede)

④ Directorio de grupo + bit setgid

Objetivo: Construir desde cero un directorio compartido donde varios usuarios puedan colaborar. Descubrirás por qué setgid es imprescindible.
Experimento clave: Crea un fichero sin setgid, mira qué grupo tiene. Activa setgid, crea otro, y compara. Esa diferencia es la razón de que todo equipo de desarrollo use setgid.
alumno@ubuntu:~$
# Sigue los 5 pasos. Empieza creando el grupo: sudo groupadd dev_clase
alumno@ubuntu:~$
